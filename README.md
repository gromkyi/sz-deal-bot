# STALZONE deal bot — почасовой поиск выгодных лотов с алертами в Телеграм

Каждый час проходит по списку ликвидных предметов, сравнивает минимальную цену покупки (лоты)
с медианой реальных сделок и шлёт в Телеграм те, где **чистая маржа после комиссии 5% ≥ 15%**
и цена покупки в рамках бюджета. Крутится в облаке (GitHub Actions) — твой компьютер не нужен.

## Как считается выгода
`чистыми = медиана_сделок × 0.95 − цена_покупки`, маржа `% = чистыми / цена_покупки`.
Пример: купил за 80k, медиана 100k → 100k×0.95 − 80k = **+15k (18.75%)**.

**Все пороги — в `config.json`** (код менять не нужно):
- `margin_tiers` — обратная прогрессия маржи: <10k → 25%, 10–100k → 20%, 100k–1M → 15%, >1M → 10%.
- `budget.min_buy` / `budget.max_buy` — диапазон цены покупки.
- `display_blocks` — на какие блоки группировать выдачу (5–50k, 50–100k, 100–500k, 500k–1M, >1M).
- `min_liquidity_per_day`, `auction_fee`, `alert_cooldown_hours`, `max_items_per_block`,
  `sort_within_block` (`profit` — по выгоде в монетах, `margin` — по %).

## Шаг 1. Собрать список ликвидных (локально, разово)
Скопируй `market_overview.csv` (из папки `price_tracker`) в эту папку `deal_bot` и запусти:
```
python build_watchlist.py
```
Получишь `watchlist.csv` с названиями. Пороги ликвидности/цены — вверху `build_watchlist.py`
(`MIN_LIQ`, `MIN_MEDIAN`, `MAX_MEDIAN`). Под бюджет ~1 млн разумно `MIN_MEDIAN=10000`, `MAX_MEDIAN=1200000`.

## Шаг 2. Телеграм-бот
1. Напиши **@BotFather** → `/newbot` → получишь **токен** вида `123456:ABC...`.
2. Напиши своему боту любое сообщение (чтобы он мог тебе писать).
3. Узнай свой **chat_id**: напиши боту **@userinfobot** — он пришлёт твой ID (число).

## Шаг 3. Репозиторий на GitHub
1. Создай **публичный** репозиторий (у публичных — безлимитные минуты Actions; секреты всё равно скрыты).
2. Залей в него содержимое папки `deal_bot`: `deal_scanner.py`, `watchlist.csv`,
   `.github/workflows/scan.yml`, `.gitignore`, `README.md`.
   ⚠️ НЕ заливай `api_credentials.json` и `telegram_config.json` (они в .gitignore).
3. В репозитории: **Settings → Secrets and variables → Actions → New repository secret** — добавь 4 секрета:
   - `STALZONE_CLIENT_ID` = `3228`
   - `STALZONE_CLIENT_SECRET` = твой секрет приложения
   - `TG_BOT_TOKEN` = токен из BotFather
   - `TG_CHAT_ID` = твой chat_id
4. Вкладка **Actions** → включи workflows → открой `stalzone-deal-scan` → **Run workflow** (ручной тест).
   Если всё ок — раз в час будет приходить алерт (только когда есть подходящие лоты).

## Локальный тест (по желанию)
Положи рядом `api_credentials.json` (уже есть формат) и `telegram_config.json`:
```json
{ "bot_token": "123456:ABC...", "chat_id": "123456789" }
```
и запусти:
```
python deal_scanner.py
```
Без телеграм-настроек скрипт просто напечатает найденные сделки в консоль.

## Важные нюансы
- **GitHub Actions cron — по UTC и иногда с задержкой** (под нагрузкой может опоздать на 5-15 мин). Для торговли это норм.
- **Минуты Actions:** один прогон ~5-8 мин. На **публичном** репозитории это бесплатно и без лимита. На приватном — есть месячный лимит, тогда уменьшай список или ставь раз в 2 часа.
- **Осторожно с оружием/бронёй/артефактами с улучшениями:** у них есть скрытые атрибуты (износ, заточка, качество), и «медиана» по ним обманчива — дешёвый лот часто дешёвый не просто так. Бот даёт наводку, но перед покупкой **проверяй предмет в игре**. Самые надёжные флипы — фунгибельные предметы (расходники, патроны, простые артефакты).
- `deals_log.csv` копит историю всех найденных сделок — потом сделаем по нему аналитику «что реально окупается».
- Антиспам: один и тот же предмет не дублируется чаще, чем раз в 6 часов (если цена не упала ещё на 5%).
